# 도메인별 질문 설계 가이드

## 사용 방법

연구 주제가 주어지면, 아래 패턴을 참고하여 **해당 도메인의 핵심 개념**에 맞는 질문을 설계합니다.
이 문서의 예시를 그대로 사용하지 않고, 연구 주제에 맞게 응용합니다.

---

## 1. 질문 설계 패턴

### 패턴 A: 개념 이해도

```
"[핵심 개념]에 대해 어느 정도 알고 계시나요?"
옵션:
  - "처음 들어봅니다"
  - "개념은 알지만 직접 다뤄본 적은 없습니다"
  - "관련 논문을 읽거나 코드를 실행해본 적이 있습니다"
  - "직접 구현하거나 연구에 활용한 경험이 있습니다"
```

### 패턴 B: 방법론 경험

```
"[방법론/알고리즘] 관련 경험이 있으신가요?"
옵션:
  - "관련 경험이 전혀 없습니다"
  - "[기초 수준 경험 예시]"
  - "[중급 수준 경험 예시]"
  - "[고급 수준 경험 예시]"
```

### 패턴 C: 데이터 유형 경험

```
"[데이터 유형] 데이터를 다뤄보신 경험이 있나요?"
옵션:
  - "이런 유형의 데이터를 다뤄본 적이 없습니다"
  - "간단한 탐색/시각화 수준으로 다뤄봤습니다"
  - "전처리와 분석을 직접 해본 경험이 있습니다"
  - "대규모 처리 파이프라인을 구축한 경험이 있습니다"
```

---

## 2. 도메인별 핵심 개념 예시

아래는 참고용 예시입니다. 실제 연구 주제에 맞게 핵심 개념을 식별하세요.

### 자연어 처리 (NLP)

| 연구 유형 | 질문해야 할 핵심 개념 |
|-----------|---------------------|
| 시맨틱 검색 | 임베딩, 벡터 유사도, 사전학습 모델, 해당 언어 전처리 |
| 텍스트 분류 | 분류 모델(BERT 등), fine-tuning, 평가 메트릭(F1 등) |
| 정보 추출 | NER, 관계 추출, 지식 그래프 |
| 텍스트 생성 | LLM, 프롬프트 엔지니어링, 디코딩 전략, 평가(BLEU 등) |

### 컴퓨터 비전 (CV)

| 연구 유형 | 질문해야 할 핵심 개념 |
|-----------|---------------------|
| 이미지 분류 | CNN 아키텍처, 전이학습, 데이터 증강 |
| 객체 탐지 | 앵커 박스, NMS, mAP, YOLO/RCNN 계열 |
| 3D 비전 | 포인트 클라우드, 메시, SDF, NeRF |
| 의료 영상 | DICOM, 세그멘테이션, 데이터 불균형 |

### 강화학습 (RL)

| 연구 유형 | 질문해야 할 핵심 개념 |
|-----------|---------------------|
| 로봇 제어 | 상태/행동 공간, 보상 설계, sim-to-real |
| 게임 AI | 환경 설계, 정책 학습(DQN, PPO), 자기 대전 |
| 최적화 | MDP 모델링, 탐색-활용 트레이드오프 |

### 데이터 사이언스

| 연구 유형 | 질문해야 할 핵심 개념 |
|-----------|---------------------|
| 예측 모델링 | 특성 공학, 교차 검증, 앙상블 방법 |
| 시계열 분석 | 정상성, ARIMA/Prophet, 시퀀스 모델 |
| 추천 시스템 | 협업 필터링, 콘텐츠 기반, 임베딩 |

### 로보틱스 / 시뮬레이션

| 연구 유형 | 질문해야 할 핵심 개념 |
|-----------|---------------------|
| 경로 계획 | SDF/ESDF, 샘플링 기반 알고리즘(RRT, PRM), 최적 제어 |
| SLAM | 점군 처리, 맵 표현, 루프 클로저 |
| 매니퓰레이션 | 역기구학, 그래스프 플래닝, 힘 제어 |

## 3. 숙련도 신호 해석

사용자의 답변에서 역량 수준을 추정하는 기준:

| 답변 신호 | 추정 수준 |
|-----------|----------|
| "처음 들어봅니다", "잘 모릅니다" | 경험 없음 |
| "개념은 알지만...", "이론적으로는 알지만..." | 개념 이해 수준 |
| "수업/튜토리얼에서 해봤다", "간단한 예제를 따라해봤다" | 입문 |
| "과제/프로젝트에서 직접 사용했다" | 초급 |
| "연구에서 활용하고 있다", "문제 해결에 사용한다" | 중급 |
| "직접 구현할 수 있다", "남에게 가르칠 수 있다" | 고급 |
