# 난이도 채점 루브릭

## 1. 점수 산출 가이드

종합적으로 아래 세 요소를 고려하여 0~100 점수를 산출합니다.
각 요소를 별도로 점수화하여 테이블로 보여주지 않습니다.
내부 판단 기준으로만 활용합니다.

### 기술 격차 (가장 큰 비중)

핵심 기술의 보유/미보유를 대조합니다.

**격차 판단 기준:**

| 상태 | 의미 | 난이도 영향 |
|------|------|-----------|
| 필요 수준 이상 보유 | 강점 | 낮춤 |
| 1단계 차이 | 소폭 학습 필요 | 약간 높임 |
| 2단계 차이 | 상당한 학습 필요 | 상당히 높임 |
| 3단계 이상 차이 | 기초부터 학습 | 크게 높임 |
| 핵심 기술 전체 미보유 | 해당 영역이 완전히 새로움 | 매우 크게 높임 |

**핵심 기술 vs 보조 기술:**
- 핵심 기술(이것 없이는 연구 불가)의 격차는 난이도에 큰 영향
- 보조 기술(대안 가능)의 격차는 영향이 작음

### 연구 복잡도 (연구 자체의 본질적 난이도)

| 수준 | 특성 |
|------|------|
| 낮음 | 기존 도구/모델을 그대로 활용, 단순 분석 |
| 보통 | 기존 모델 fine-tuning, 중규모 데이터, 여러 도구 연동 |
| 높음 | 커스텀 모델 설계, 대규모 실험, 멀티모달 처리 |
| 매우 높음 | 새로운 아키텍처, 분산 학습, 최적화 연구 |

### 환경 요인

- **도구 통합**: 사용해야 할 도구/서비스 간 연동 복잡도
- **인프라**: GPU, 클라우드, 특수 환경 구축 난이도
- **프로젝트 규모**: 예상 기간, 코드량, 실험 수

## 3. 학습 권장 우선순위

| 우선순위 | 의미 | 기준 |
|----------|------|------|
| 🔴 필수 선행 | 연구 착수 전 반드시 학습 | 핵심 기술이면서 격차가 큰 것 |
| 🟡 병행 학습 | 연구하면서 함께 학습 가능 | 중요하지만 바로 필요하지 않거나, 격차가 작은 것 |
| 🟢 후순위 | 필요할 때 배워도 됨 | 보조 기술이거나, 대안이 있는 것 |

**학습 권장 작성 원칙:**
- 각 항목에 **이 연구에서의 구체적 용도**를 명시
- **예상 학습 기간**을 현실적으로 제시
- 가능하면 **구체적 학습 자원**(공식 튜토리얼, 특정 강좌 등)을 권장
- 총 3~5개 항목이 적당 (너무 많으면 압도적)
