# 타당성 판정 세부 기준

## 1. 기술 성숙도 분류

| 단계 | 정의 | 판정 반영 |
|------|------|-----------|
| **연구 단계** | 논문에서만 제안, 공개 구현 없음 | ❌ 또는 ⚠️ (직접 구현 필요) |
| **실험적** | 오픈소스 구현 존재하나 안정성 미검증, 문서 부족 | ⚠️ |
| **안정** | 프로덕션 사용 사례 존재, 활발한 유지보수, 문서 충분 | ✅ |
| **성숙** | 업계 표준, 대규모 커뮤니티, 풍부한 생태계 | ✅ |

---

## 2. 연구 분야별 주요 기술 현황

### 2.1 자연어 처리 (NLP)

| 태스크 | 대표 기술/모델 | 성숙도 | 비고 |
|--------|--------------|--------|------|
| 텍스트 분류 | BERT, RoBERTa, fine-tuning | 성숙 | Hugging Face로 간편 사용 |
| 생성 | GPT-4, Claude, LLaMA | 성숙 | API 또는 로컬 실행 |
| 정보 추출 | spaCy, NER 모델 | 성숙 | 도메인 특화 시 학습 필요 |
| 질의응답 | RAG, Vector DB + LLM | 안정 | LangChain, LlamaIndex |
| 다국어 | mBERT, XLM-R | 안정 | 언어별 성능 차이 존재 |

### 2.2 컴퓨터 비전 (CV)

| 태스크 | 대표 기술/모델 | 성숙도 | 비고 |
|--------|--------------|--------|------|
| 이미지 분류 | ResNet, EfficientNet, ViT | 성숙 | pretrained 모델 풍부 |
| 객체 탐지 | YOLO, DETR | 성숙 | 실시간 가능 |
| 세그멘테이션 | SAM, Mask R-CNN | 안정 | SAM은 zero-shot 가능 |
| 생성 | Stable Diffusion, DALL-E | 안정 | 학습에 고사양 GPU 필요 |
| 3D 재구성 | NeRF, 3D Gaussian Splatting | 실험적 | 빠르게 발전 중 |

### 2.3 데이터 사이언스 / 통계

| 태스크 | 대표 도구 | 성숙도 | 비고 |
|--------|----------|--------|------|
| 탐색적 분석 | pandas, matplotlib, seaborn | 성숙 | Python 생태계 표준 |
| 통계 검정 | scipy, statsmodels, R | 성숙 | |
| 시계열 분석 | Prophet, statsmodels, ARIMA | 안정 | |
| 인과 추론 | DoWhy, EconML | 실험적~안정 | 도구는 존재하나 이해 필요 |
| 베이지안 분석 | PyMC, Stan | 안정 | 학습 곡선 높음 |

### 2.4 강화학습 / 시뮬레이션

| 태스크 | 대표 도구 | 성숙도 | 비고 |
|--------|----------|--------|------|
| 기본 RL | Gymnasium, Stable-Baselines3 | 안정 | |
| 멀티 에이전트 | PettingZoo, RLlib | 실험적~안정 | |
| 로보틱스 시뮬레이션 | MuJoCo, Isaac Sim | 안정 | GPU 필요 |
| 커스텀 환경 | Gymnasium API | 안정 | 직접 구현 필요 |

### 2.5 생물정보학 / 의료

| 태스크 | 대표 도구 | 성숙도 | 비고 |
|--------|----------|--------|------|
| 유전체 분석 | Biopython, GATK | 성숙 | |
| 단백질 구조 | AlphaFold, ESMFold | 안정 | 사전학습 모델 공개됨 |
| 의료 영상 | MONAI, 3D Slicer | 안정 | 데이터 접근성 이슈 |
| 약물 발견 | RDKit, DeepChem | 안정 | 도메인 지식 필수 |

## 3. 데이터 확보 가능성 체크리스트

| 데이터 유형 | 확인 항목 |
|------------|-----------|
| **공개 데이터셋** | Kaggle, HuggingFace Datasets, UCI ML Repository, Papers with Code에 존재하는지 |
| **API 수집** | 해당 서비스가 API를 제공하는지, rate limit과 비용은 어떤지 |
| **웹 크롤링** | robots.txt 허용 여부, 법적 이슈, 데이터 규모 |
| **자체 생성** | 실험/설문/시뮬레이션으로 생성 가능한지, 소요 시간 |
| **기관 제공** | 학교/연구소/병원 등에서 접근 가능한 데이터가 있는지 |
| **합성 데이터** | 실제 데이터가 불가할 경우 합성 데이터로 대체 가능한지 |

## 4. 컴퓨팅 자원 현실성 기준

| **HPC 클러스터** | 대학/연구소 제공 | $0 (소속 시) | 대규모 실험 |
| **풀 클라우드** | 멀티 GPU, 대용량 스토리지 | $100+/월 | 대규모 프로덕션 |
